조사 팩 마커를 명시적으로 갖춘 형식으로 다시 정리합니다.

**재현:** `python demo.py` → 기대 `2 + 3 = 5`, 실제(수정 전) `2 + 3 = -1`.

**가설 1:** `add` 함수 본문의 연산자 오타 — 덧셈이어야 할 곳이 `a - b`(뺄셈).
**증거:** `calc.py:2`가 `return a - b`. 함수명 `add`, 호출부 `demo.py`의 라벨 `"2 + 3 ="` 모두 덧셈 의도. 연산자만 `-`. 수정 후 `a + b`로 재실행 시 `5` 출력 → 인과 확정.

**가설 2:** 의도는 뺄셈인데 함수 이름만 잘못 지음.
**기각:** `demo.py`가 `2 + 3` 라벨로 결과를 출력하므로 호출부가 덧셈을 기대함 → 이름이 아니라 연산자가 틀림.

**가설 3:** `add`가 다른 모듈에서 재정의/오버라이드됨.
**기각:** `add` 정의는 `calc.py` 한 곳뿐, `demo.py`는 그대로 import. 경쟁 정의 없음.

**전후 검증:**
- 수정 전: `return a - b` → `add(2,3) = -1`
- 수정 후: `return a + b` → `python demo.py` 실행 결과 `2 + 3 = 5` ✅

`calc.py`의 `a - b` → `a + b` 한 줄 수정으로 해결 완료했습니다.
