이 팩이 주입됐다는 것은 이번 요청이 모호하다고 이미 판정됐다는 뜻이다. 모호한지 아닌지를 네가 다시 판단할 필요는 없다 — 지금 할 일은 작업을 바로 시작하는 게 아니라 의도부터 확정하는 것이다.

## 1. 무엇을 물을지 정하라 — 이미 명확한 건 빼라

아래 3개 슬롯 중 **프롬프트로 이미 명확한 것은 묻지 마라.** 정말 불확실한 것만 질문 대상이다:
- **목표**: 최종적으로 뭘 원하는가
- **범위**: 어디를(어떤 파일·기능·영역을) 건드려도 되는가
- **비목표**: 뭘 건드리면 안 되는가 / 안 해도 되는가

## 2. 하나씩 물어라(1문1답) — 객관식 우선, 메타발언 금지

### 판정 규칙
1. 위 3개 슬롯 중 아직 불확실한 게 있고, 지금까지 던진 질문이 3개 미만인가? 아니면(다 명확하거나 이미 3개 물었으면) 3단계로 가라.
2. 불확실한 것 중 **하나만** 골라 `확인질문 N:`으로 묻고, 그 질문(물음표로 끝나는 문장)으로 턴을 끝내라. 여러 개를 한 번에 나열하지 마라 — 한 번에 하나씩이다.
3. 사용자 답을 받으면 1로 돌아가 다시 판정하라.

질문은 서두("질문 좀 드릴게요", "확인이 필요합니다" 같은 메타 서술) 없이 바로 시작하라:
```
확인질문 N: [질문 내용]
```
비개발자도 바로 답할 수 있도록 **객관식을 우선**하라 — "A로 할까요, B로 할까요?"처럼. 객관식으로 만들기 어려울 때만 개방형으로 물어라.

## 3. 답이 다 모이면 말로 확인하지 말고 곧장 intent set을 실행하라

"확인했습니다", "정리하면 이렇습니다"처럼 답을 말로 재서술하지 마라 — 그 자체가 군더더기다. 필요한 질문에 답이 다 모이면(또는 처음부터 더 물을 게 없었으면) **훅 컨텍스트나 차단 메시지에 표시된 `fable-lite-cli.py` 절대경로 명령을 그대로 tool로 호출**하라. 명령의 `--goal`, `--scope`, `[--non-goal]` 자리만 확정된 값으로 채운다. 저장소 루트에서 직접 작업 중일 때는 `python -m fable_lite intent set ...`도 가능하지만, 플러그인 설치 상태에서는 표시된 절대경로 명령을 우선한다.
`--scope`엔 파일 경로(글롭 가능, 예: `"src/auth/**"`)나, 아직 파일을 특정할 수 없으면 답변에서 나온 자연어 영역(예: `"로그인 폼"`)을 그대로 넣어도 된다.
이 명령 실행이 곧 확정이다 — 실행 전까지는 의도가 미확정 상태라 Edit/Write류가 계속 차단된다. Bash로 조사·재현하는 건 의도 확정 전에도 막히지 않으니, 질문을 줄이기 위해 먼저 코드를 살펴보는 것도 좋다.

## 4. 사용자가 "그냥 해"/"알아서 해"라고 하면: 가정을 선언하고 진행하라

되묻지 마라. 대신 네가 채택할 가정을 명시적으로 선언하라:
```
가정 N: [네가 채택한 가정]
```
그다음 `--assumed` 플래그를 붙여 기록하고 진행하라:
```
<컨텍스트/차단 메시지에 표시된 정확한 intent set 명령> --assumed
```
가정은 나중에 틀렸다고 밝혀지면 정정할 수 있도록, 무엇을 가정했는지가 기록에 분명히 남아야 한다 — `가정 N:` 없이 조용히 넘어가지 마라.

---

### 예시 (전체 흐름 — 하위 모델은 이 패턴을 그대로 따라 하라)

**사용자**: "이거 좀 고쳐줘"

**어시스턴트** (1턴 — 질문 1개만 던지고 종료):
```
확인질문 1: "이거"가 어느 파일/기능을 말하는 건가요? (예: 방금 얘기하던 로그인 폼인가요, 다른 곳인가요?)
```

**사용자**: "로그인 폼이요, 제출 버튼 누르면 아무 반응이 없어요"

**어시스턴트** (2턴 — 범위는 이제 명확해졌으니 비목표 하나만 더 질문):
```
확인질문 2: 동작만 고치면 될까요, 아니면 디자인(스타일)도 같이 손봐도 될까요?
```

**사용자**: "동작만요. 디자인은 건드리지 마세요"

**어시스턴트** (3턴 — 답이 다 모였으니 말로 재확인하지 않고 바로 tool 호출):
```
<컨텍스트/차단 메시지에 표시된 fable-lite-cli.py 절대경로 명령을 사용해> --goal "로그인 폼 제출 버튼 무반응 수정" --scope "로그인 폼" --non-goal "스타일/디자인 변경"
```
(이후 investigation 팩 절차대로 재현·조사를 진행한다.)

---

### 게이트 계약 (PreToolUse)

`.fable-lite/intent.json`이 없는 상태로 Edit/Write/MultiEdit/NotebookEdit을 시도하면 PreToolUse가 "의도가 확정되지 않았습니다" 사유로 차단한다 — 위 3단계(질문 완료 후 intent set)나 4단계(가정 선언 후 --assumed) 중 하나를 실행해야 풀린다. 차단은 최대 2회 후 통과한다(무한 트랩 금지). Bash는 이 게이트의 대상이 아니다 — 조사·재현은 의도 확정 전에도 자유롭게 하라.
