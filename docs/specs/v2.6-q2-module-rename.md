## 1. 목표
- 파이썬 패키지 canonical 이름을 `smtw`로 통일 (`fable_lite/` → `smtw/`).
- `fable_lite`는 **deprecation shim**으로 유지 — 기존 호출(`import fable_lite`, `python -m fable_lite ...`) 동작 유지.
- SemVer **minor (v2.6.0)**. "breaking 0"의 정의: **기본 경고 필터 기준** 동작 불변. `-W error::DeprecationWarning`·`PYTHONWARNINGS=error` 강제 환경은 호환 보장 대상에서 제외(경고를 에러로 승격하는 것은 사용자의 명시적 선택) — CHANGELOG·README에 명시. shim 제거는 v4(Q5 확정 조항).

## 4. 수용기준 (GWT)
1. **[canonical]** `python -m smtw <cmd>` 전 서브커맨드가 v2.5.0과 동일 동작(계약=exit code·stdout JSON·기능).
2. **[shim-모듈]** `python -m fable_lite <cmd>`/`import fable_lite` 동일 동작 + DeprecationWarning 정확 1회/프로세스(패키지+복수 서브모듈 import 시에도 1회) + 훅 stdout JSON 무오염(파싱 테스트).
3. **[shim-identity]** `fable_lite is smtw`, `fable_lite.cli is smtw.cli`(전 서브모듈), `importlib.reload` 후에도 유지, Windows multiprocessing spawn 자식에서도 유지, 구버전 pickle fixture 역직렬화 성공.
4. **[패키징·클린 게이트 (codex C-02·C-03, agy H-02)]** 절차를 hard gate로: ①검증된 저장소 경로의 `build/`·`dist/`·`fable_lite.egg-info/`·`__pycache__` 명시 정리 ②clean build ③**wheel RECORD 검사**(`smtw/**` 전체 + shim `fable_lite/__init__.py`만 존재) ④egg-info `entry_points.txt` 2종 모두 `smtw.cli:main` ⑤격리 venv 설치 후 identity·콘솔 스크립트 검증. **구현·게이트 전제=clean 상태**(영진의 stale 2.4.1 설치 메타데이터로 인한 기존 1fail은 사전 정리 — 혼재 설치는 게이트 전제 위반으로 정의, version 조회 로직 현행 유지).
5. **[게이트]** pytest 전체 green + ruff + compileall + probes + **adapters E2E replay**(구버전 안내 문자열 호환 실증) green — 로컬 게이트=CI 게이트 정합.
6. **[릴리스]** CI green 머지 → tag v2.6.0 + Release + release-quality green + 플러그인 2.6.0 갱신.
7. **[strict-warning 정책]** `-W error::DeprecationWarning`에서 `import fable_lite`가 에러가 되는 것은 **의도된 동작**(§1) — 테스트로 고정하고 CHANGELOG에 명시.
8. **[경고 필터 실증 (codex 2R critical)]** pytest subprocess 테스트로 양쪽 실증: ①기본 필터+§2-3 filterwarnings 상태에서 shim import 포함 pytest green(경고가 게이트를 깨지 않음) ②`PYTHONWARNINGS=error` 주입 시 shim import가 에러로 승격됨(=§4-7 정책 일치).
