# v2.2 — Quiet Opt-in (실업무 방해 해소)

> 2026-07-17 3-AI 설계 수렴(Claude Fable 5 오케스트 + Codex gpt-5.6-sol max 적대검토 + Antigravity Gemini 3.1 Pro 적대검토).
> 문제: 훅이 모든 세션·디렉토리에 상시 개입 — 홈 `.fable-lite/` 오염, 도구호출당 Python 콜드스타트 2회, 정보성 systemMessage 스팸, cwd 기반 project_root 오판(2026-07-16 R1×2+Stop×2 오차단 실사고).
> 범위: **claude_code 어댑터 한정.** shared core의 Codex/agy 어댑터 동작 불변.

## A1. Opt-in 활성화 게이트

- 활성화 신호: `<root>/.fable-lite/config.json` = `{"schema_version": 1, "supervision": true}`.
  - `true`는 **정확한 boolean만** 인정. absent/false/비boolean = 비활성.
  - `.fable-lite/` 디렉토리·contract 존재만으로 자동 활성화 **금지** (홈 오염 이력 때문).
  - 손상된 config = 세션당 1회 건강 경고 후 비활성 취급.
- 판정 시점: **UserPromptSubmit에서 1회 결정 후 세션 동안 latch.** 활성 세션 중 config 삭제·변경으로 비활성화되지 않음 (자기 감독 끄기 방지).
- exact home(사용자 홈 디렉토리 그 자체)은 config 있어도 **하드 오프**. 홈 하위 정상 프로젝트는 허용.
- 게이트 위치: 어댑터 `main()` 최상단, **stdlib만으로** stdin payload 파싱→루트 판정→게이트. 비활성이면 core import 0·파일 생성 0·원장 접근 0·출력 정확히 `{}`.
  - ⚠️ 현재 결함: `user_prompt_submit.py`가 홈에서도 `record_event()`로 원장을 생성함. 홈/비활성 차단은 **모든 상태 접근보다 앞**에 둘 것.
- 한계 명시: 전역 플러그인 설치에서는 Python 인터프리터 기동 비용은 남는다(“콜드스타트 0” 아님 — “import·상태·메시지 0”). 진짜 0은 프로젝트 scope 설치(운영 문서 D 참고).

## A2. 프로젝트 루트 고정 (cd 함정 제거)

- 우선순위: **`CLAUDE_PROJECT_DIR` env(훅 실행 시 Claude Code가 제공) → 세션 레지스트리 latch → (레지스트리 유실+env 부재 시에만) cwd 상향탐색 폴백.**
- 세션 레지스트리: `<플러그인 데이터 디렉토리>/sessions/<sha256(session_id)>.json`
  - (구현자: `CLAUDE_PLUGIN_DATA` 등 공식 데이터 경로 env 실측 확인, 부재 시 플러그인 루트 밖 사용자 홈 하위 전용 디렉토리 사용. 프로젝트 원장과 절대 혼합 금지)
  - 기록: schema_version, canonical root, config digest, 생성·최종활동 시각.
  - **atomic write / write-once.** 경로 mismatch 시 덮어쓰기·재바인딩 금지. 고정 후 cwd 폴백 금지.
  - 파일 유실 시 `CLAUDE_PROJECT_DIR`와 활성 config가 **모두 일치할 때만** 재구성.
  - GC: SessionEnd 훅에서 정리 + UserPromptSubmit에서 TTL 기반 잔여 청소 (비정상 종료 대비).
- 레지스트리는 **루트 전달용 adapter 상태일 뿐** — host/session/agent 다중 에이전트 identity와 분리. turn_id는 가능하면 훅 입력의 `prompt_id`, identity는 `agent_id` 반영.
- `/add-dir` 외부 경로는 v2.2 감독 범위 밖(문서화).
- 참고 결함(수정 대상): `common.py:project_root()`의 containment 검사가 역방향 — probe에서 cwd=fable-lite/core, project_root=fable-lite 입력 시 fable-lite/core 반환.

## A3. 메시지 다이어트

- 출력 정책:
  - 비활성 프로젝트·exact home·정상 관측·정상 Stop allow → **정확히 `{}`**.
  - block → 구조화된 차단 사유 표시 (유지).
  - scope warning·intent 인터뷰 컨텍스트 → **turn당 1회**. scope warn의 systemMessage+additionalContext 이중 출력 제거.
  - 건강 이상(원장 손상·세션 루트 불일치·fail-open 예외) → **세션당 1회** 경고. ⚠️ fail-open 무음화 금지 (감독이 죽었는데 살아있다고 믿게 만들면 안 됨 — 양 리뷰어 공통 지적).
  - scorecard 라인은 원장/CLI에만 보존, 표시는 별도 opt-in.
- 정보성 메시지(observed N/provenance incomplete/검증 기록/Stop gate allow/HOME advisory)는 전부 무음화하되 **원장 기록은 유지**.
- Stop의 assistant text는 훅 입력 `last_assistant_message` 우선, transcript 파싱은 구버전 fallback.

## A4. quick 모드 경량화 (관측 유지 + 실행이력 기반 승격)

- **PostToolUse/PostToolUseFailure 관측은 어떤 모드에서도 유지** (변경 귀속·invocation close·contract Pre/Post 이중 증거·롤백 근거 = 멀티에이전트 핵심 불변조건. 실패한 shell도 부분 변경 가능).
- quick 경량화는 lazy-baseline 방식:
  - quick 시작 시 eager baseline·전체 scan·메시지 생략.
  - **최초 mutation-capable(또는 unknown) 도구 직전에 normal로 원자적 승격** — baseline·R2·contract 검사 수행 후 도구 허용. (classify의 quick 판정은 휴리스틱+기본값이라 안전 경계로 신뢰하지 않는다)
  - 실제 이력상 mutation-capable 호출·known change가 전무한 턴만 무거운 Stop reconcile 생략, 이때도 "clean verified" 주장 없이 allow.
  - 변경 가능성 발견 시 기존 `verify_state.evaluate_without_io` 정상 게이트 적용.

## A5. 호환성·기타 계약 정리

- opt-in 정책은 claude_code 어댑터에만. shared core·codex_cli·antigravity 어댑터 불변.
- 루트 bootstrap 후 프로젝트 원장 접근은 기존 R2-first 순서 유지. config/세션 레지스트리 읽기는 "adapter bootstrap 예외"로 명시(프로젝트 원장 접근과 구분).
- PreToolUse의 top-level `decision:"block"`은 deprecated → `hookSpecificOutput.permissionDecision:"deny"`로 전환.

## A6. 테스트 계약

- 기존 R2·verification epoch·contract 이중 증거·멀티에이전트 격리 테스트는 **삭제·약화 금지**.
- 기대 출력이 바뀌는 테스트(예: `tests/test_adapters.py`의 config 없는 tmp 프로젝트 활성화·정보 메시지 assertion)는: adapter fixture에 명시적 활성 config 추가 + 메시지 assertion을 `{}` + 원장 이벤트 assertion으로 교체.
- 테스트 하네스 강제 활성화: env `SMTW_TEST_FORCE_ENABLE=1` (테스트 전용 우회, 프로덕션 경로에선 문서화된 테스트 스위치임을 명시).
- 신규 회귀 테스트: 비활성 프로젝트 무상태 / exact home 무상태 / nested cwd 루트 고정 / config 변경 latch / 레지스트리 손상·GC·재구성 / 동시 세션·subagent / quick 오분류 승격 / 실패 후 부분 변경.

## 수용 기준 (측정 가능)

1. 비활성 프로젝트(프로젝트 scope 설치 시): 훅 프로세스 실행 0.
2. 전역 설치 fallback: core import 0 · 파일 생성 0 · 출력 `{}` (실측: 비활성 디렉토리에서 훅 실행 후 `.fable-lite/` 미생성 확인).
3. quick read-only 턴: scan·상태 쓰기·Stop reconcile 0.
4. quick 중 mutation(또는 실패 후 부분 변경): normal과 동일한 귀속·계약·검증 차단.
5. `python -m pytest tests -q` green (기존 불변식 유지 + 신규 회귀 포함).
