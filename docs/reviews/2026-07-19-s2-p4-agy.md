## 1. R2 Identity Laundering 및 Journal I/O Fail-Open (SCORE-1)
- **우려 사항**: R2 차단 시점의 에이전트 식별자가 미정산(합성/날것) 상태라 `coordination.jsonl`에 성적 세탁이 발생할 수 있는지, 또한 저널 쓰기(I/O) 실패 시 Fail-Open 처리가 되어 차단(Block) 결정이 통과(Allow)로 세탁되는지.
- **실측 결과**: **안전함**
  - **파일:라인 근거**: 
    - `adapters/claude_code/pre_tool_use.py:155-179` 
    - `core/adapter_observation.py:286-341` (`record_r2_deny_after_resolution`)
  - **분석**: R2 게이트 평가(`evaluate_r2_destructive_gate`)는 날것의 `invocation`으로 빠르고 엄격하게 수행합니다. 만약 "차단(Block)" 결정이 내려지면, 반환 직전에 `record_r2_deny_after_resolution()`을 호출하여 `resolve_active_invocation()`을 통해 *진짜 소유자 신원*을 확정한 뒤 저널에 기록합니다( split-brain 방지). 
  - 가장 중요한 안전장치로, `record_r2_deny_after_resolution` 내부의 `try_record_coordination_event`가 파일 시스템 권한 오류 등으로 실패(예외 발생)하더라도, `except Exception`으로 예외를 삼키고 `False`를 반환할 뿐입니다. 이 반환값은 무시되며, 오케스트레이터 훅은 **원래의 Block 결정을 그대로 사용자에게 반환(`_deny`)**합니다. 즉, 관측(Logging)은 Fail-Open이지만 통제(Gate)는 Fail-Closed 원칙을 엄격하게 준수합니다.

## 2. F3 Peer Activity Recovery 버그 (FIX-1)
- **우려 사항**: `resume_turn` 중 `COMPLETE` 엄격 비교 버그로 인해 F3(타 에이전트 충돌 배제) 턴 복구가 실패하는지.
- **실측 결과**: **해결됨**
  - **파일:라인 근거**: `core/provenance_lifecycle.py:309`
  - **분석**: 기존의 `result.status is ProvenanceStatus.COMPLETE` 엄격 비교 코드가 `_complete_observation(result.status)` 호출로 수정되었습니다. 해당 헬퍼 함수는 `COMPLETE`뿐만 아니라 `COMPLETE_WITH_EXCLUSIONS`(F3 조건)까지 허용하므로, 부트스트랩 윈도우에서의 Peer Activity 복구 버그가 완벽히 고쳐졌습니다.

## 3. "Turn Not Started" 좀비 세탁 방지 (FIX-2)
- **우려 사항**: `begin_invocation` 시 턴 상태가 좀비로 남아있다가, Fail-Open 시 실패한 부트스트랩을 강제로 `COMPLETE`로 세탁하여 게이트를 약화시키는지.
- **실측 결과**: **안전함**
  - **파일:라인 근거**: `core/adapter_observation.py:142-164` (`begin_invocation`)
  - **분석**: `lifecycle.begin_invocation` 내부에서 `TurnBootstrapError` 예외가 발생하면 즉시 `except` 블록으로 빠져나와 `_record_status`로 실제 에러 상태(Incomplete)를 기록하고 조기 반환합니다. `_record_invocation` 호출(여기서 `COMPLETE` 기록)은 **성공적으로 관측이 완료된(예외가 발생하지 않은) 경우에만** 도달하게 설계되어, 실패를 성공으로 세탁하는 취약점(FIX-2)이 완전히 차단되었습니다.

## 4. 경로 정규화 우회: Windows Long Paths (FIX-3)

- **실측 결과**: **안전함**
  - **파일:라인 근거**: `adapters/claude_code/atomic_file.py:30-42` (`_atomic_paths`)
  - **분석**: `_needs_extended_path`를 검사하고 `\\?\` 접두사를 붙이기 **전에**, 먼저 `os.path.normpath(os.path.abspath(...))`를 호출합니다. 이로 인해 모든 디렉토리 탐색(Traversal) 연산자(`.`, `..`)가 절대 경로로 안전하게 정규화(해소)된 이후에만 문자열 접두사가 추가됩니다. 정규화 우회 공격이 불가능합니다. (게다가 해당 로직은 `atomic_file.py`로, .claude 내부 세션 레지스트리에만 쓰여 유저 코드 파괴 벡터가 아님).
