## 1. ACT-01 정책 A 구현 점검 결과

### 1) 비활성 B에 상태 파일 생성 여부
- **생성되지 않음 (안전)**: `env_root`(B)가 opt-in 설정이 되어있지 않아 비활성화(`enabled=False`)로 판정될 경우, 즉시 `_inactive()`를 반환합니다. 
- 테스트 `test_unconfigured_mismatch_root_remains_project_stateless`를 통해 B 디렉터리 내부에 어떤 파일(ledger, snapshot, registry)도 생기지 않음(`list(project_b.rglob("*")) == []`)이 확실히 입증되었습니다.

### 2) 무상태 fast-path 계약 유지 여부
- **유지됨 (안전)**: `_config_state`는 Python 표준 라이브러리(`os.path`, `json`)만을 사용하여 파일의 존재 유무와 내용만 빠르게 읽고 반환하는 무상태(stateless) 함수입니다. 
- 무거운 초기화 과정이나 파일 I/O(상태 저장)가 수반되지 않으므로 quiet opt-in의 fast-path 성능 계약을 깨지 않습니다.

### 3) Config 검사 우회 표면 (경로 위장)
- **우회 불가 (안전)**: `_same_root` 함수에서 `os.path.normcase(str(left.resolve()))`를 사용하여 두 경로를 비교합니다.
- `resolve()`가 심볼릭 링크를 끝까지 추적하고 경로 구분자(후행 슬래시 포함)를 정규화하며, `normcase()`가 대소문자를 일치시키므로 어떠한 문자열 꼼수로도 B를 A처럼 위장할 수 없습니다.

### 4) 경고 Dedup(1회) 동작
- **정상 동작 (안전)**: `warn_once(data_dir, session_id, "root_mismatch")`를 통해 경고 메시지를 필터링하며, 테스트 `test_root_mismatch_warning_is_emitted_once`에서 동일 세션 내 두 번째 호출 시 경고가 발생하지 않음이 확인되었습니다.

### 5) v2.2 `promote_quicklatch` 원자성과의 상호작용 회귀
- **회귀 없음 (안전)**: B가 활성화되어 B 훅으로 동작할 때도 `bind_session(..., fixed_root, ...)`을 호출하여 registry 파일은 기존 `fixed_root`(A)를 대상으로 업데이트됩니다.
- `bind_session` 내부적으로 기존의 `_path_lock`과 `_atomic_write`를 그대로 사용하므로 원자성(atomicity) 훼손이 없으며, 레지스트리가 A로 유지되므로 fast-path(`quicklatch` 동작)가 A에 대해 정상적으로 계속 작동합니다.
