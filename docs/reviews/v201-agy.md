# v2.0.1 신뢰성 패치 적대적 리뷰 보고서 (Antigravity)

## 리뷰 개요
- **대상**: v2.0.1 신뢰성 패치 (커밋 `6832460` ~ `0b93808`, 총 17개)
- **초점**: 오탐/미탐 양방향 리스크 검증, 우회로 점검, 문서 과장 여부
- **결과**: 치명 결함(Critical/Major) 0건, 모든 구현이 명세 및 설계 철학과 일치함.

## 검증 항목별 상세 리뷰

### 1. Quick 면제 제거 (Pass)
- `core/verify_state.py`의 `evaluate_without_io`에서 `quick` 모드 분기가 완전히 제거됨을 확인했습니다.
- **오탐 리스크**: 기존 `docs_only` 및 `not changed` 판단 로직이 유지되어, 단순 질문 턴이나 문서 변경 시 불필요한 검증을 요구하는 오탐이 발생하지 않음을 테스트와 코드로 실증했습니다.

### 2. Shlex 검증 판정 (Pass)
- `core/verification.py`에서 `shlex(posix=False)`를 사용하여 Windows 백슬래시 경로 및 한글 경로를 안전하게 파싱합니다.
- **미탐/오탐 반전**:
  - `echo pytest` 같은 출력 전용 명령이나 `||`, `&&` 등의 셸 연산자가 포함된 우회 시도를 원천 차단합니다.
  - `bash -c "echo pytest"`, `python -c "print('ok')"`와 같은 간접 실행 우회를 AST 및 재귀 호출로 완벽히 방어합니다.
  - 유효한 검증 명령(`npm run test`, `pytest` 등)이 정상적으로 승인되는 것을 확인했습니다.

### 3. `scope_too_large` 완화 및 홈 루트 오탐 방지 (Pass)
- 대용량 루트(예: `C:\`)에서 발생하는 `SCOPE_TOO_LARGE`가 `incomplete` 오류(차단)로 처리되지 않도록 분리되었습니다.
- 관측 불가 상태에서 알려진 변경이 없을 경우에만 '1회 advisory allow'를 부여하여 교착 상태를 해소합니다.
- **우회 우려**: 사용자가 고의로 홈 루트에서 작업할 경우 검증 게이트가 무력화되는 기술적 우회가 성립하나, 이는 도구의 스캔 한계를 반영한 정직한 동작이며, 의도적 회피 방어보다는 정직한 사용자의 작업 규율 강화라는 도구 목적에 부합합니다.

### 4. 원격 Epoch (Pass)
- `core/shell_command.py`에서 `is_remote_mutation_command`가 `ssh`, `scp` 업로드만을 정확히 포착합니다.
- **과차단 방지**: `git fetch/push` over ssh, `scp` 다운로드, 읽기 전용 SSH 옵션 등은 철저히 원격 돌연변이로 집계되지 않게 필터링되었습니다.
- **미탐 리스크**: 원격 서버 조작 후 *로컬* 검증(예: `pytest`)만으로도 원격 epoch가 풀릴 수 있습니다. (검증 명령의 목적지가 로컬인지 원격인지 정적 분석이 불가하기 때문). 이는 명세와 일치하는 의도적 한계로 수용 가능합니다.

### 5. README / CHANGELOG 과장 0 검증 (Pass)
- 문서에 구현되지 않은 기능이 기술되었거나 한계가 은폐되었는지 교차 검증했습니다.
- 원격 epoch가 로컬 테스트만으로도 풀릴 수 있다는 점, 고의적 우회를 완전히 막을 수 없다는 점, `SCOPE_TOO_LARGE` 시의 행동 등 모든 제약사항이 README와 CHANGELOG에 팩트 기반으로 정직하게 기재되어 있음을 확인했습니다.

## 총평
이번 패치는 게이트의 신뢰성을 크게 향상시켰으며, 의도치 않은 우회로나 과차단 문제를 발생시키지 않습니다. (심각도: 0)
