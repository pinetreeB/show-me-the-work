## 2. 상세 검증 결과

### [APPROVE] Chart 문맥 특별 면제 완전 제거
* **결과:** 취약점 원천 소멸 및 정상 작동
* **상세 내용:** 
  * `_chart_data_color`, `JS_LITERAL_OR_COMMENT_RE` 및 괄호 카운터(depth) 등 Chart 문맥을 추론하려 시도했던 불안정한 렉서/파서 코드가 완전히 삭제되었습니다(잔여 참조 0건).
  * 이에 따라 이전 재검증들에서 지적되었던 **모든 우회 표면(주석 내 괄호 삽입 `// [`, 정규식 리터럴 `/\/*/` 코드 삼킴, 중첩 템플릿 리터럴 악용 등)이 존재 이유를 상실**했으며, 완벽히 무력화되었습니다.
  * 이제 `.js, .ts, .jsx, .tsx` 등 스크립트 파일 내에 존재하는 Raw Color(hex, rgb, hsl)는 문맥에 관계없이 무조건 `design/raw-color` 위반으로 정확히 검출됩니다.
  * 정당하게 허용되어야 하는 Chart 데이터 색상은 기존 시스템의 원칙에 맞게 `gate.config`의 `allowlist`(경로 + 규칙 + 사유 + 만료일)를 통해서만 안전하고 투명하게 예외를 인정받도록 잘 교체되었습니다.

### [APPROVE] 기존 방어막 및 수리 항목(F2, D1~D5) 유지 확인
* **결과:** 부작용 없이 완벽 유지됨
* **상세 내용:**
  * **F2 (리팩토링 과차단 방어):** `SequenceMatcher`를 통한 `equal` 블록 면제 로직은 전혀 건드리지 않았습니다. 따라서 3차 수리에서 달성했던 "코드의 내용 변경 없이 단순 이동(Move/Refactoring)된 Legacy 코드는 과차단하지 않되, 조작/수정된 위반은 정확히 차단한다"는 이상적인 동작이 그대로 유지되고 있습니다.
  * **D1~D5:** `gate.config` 설정 파일 변경 시 즉각 PASS 상태를 무효화하는 자가면제(Laundering) 방어, 1px/0px 허용 처리, CSS 변수 및 Shorthand 정규식 확장 등 다른 하드게이트 로직들도 모두 훼손 없이 견고하게 작동합니다.
  * 로직 단순화로 인한 새로운 정규식 오탐이나 미탐은 발견되지 않았습니다.
